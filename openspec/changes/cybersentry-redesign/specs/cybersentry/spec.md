# CyberSentry Module 3 Specification

## Purpose
Transform Module 3 from Deepfakes/Vishing/Disinformation to CyberSentry focusing on criminal recruitment, money muling, cyber-extortion, and psychological manipulation for adolescents 14-20.

## Requirements

### Requirement: Criminal Recruitment Detection
The system MUST present scenarios showing criminal recruitment tactics in gaming/Discord platforms.

#### Scenario: Gaming Platform Recruitment
- GIVEN a user is in Activity 1
- WHEN the system displays a gaming platform recruitment scenario
- THEN the user can identify red flags (urgent money offers, vague job descriptions, pressure tactics)
- AND the user selects the correct response from options
- AND points are awarded based on accuracy

### Requirement: Money Mule Identification
The system MUST present scenarios showing money mule schemes via TikTok/Telegram.

#### Scenario: Social Platform Money Mule
- GIVEN a user is in Activity 2
- WHEN the system displays a money mule scenario on social platform
- THEN the user can identify warning signs (requests to receive/transfer money, new "job" offers)
- AND the user selects the correct action
- AND points are awarded based on accuracy

### Requirement: Cyber Extortion Response
The system MUST present scenarios showing sextortion and cyber-extortion attempts.

#### Scenario: Sextortion Attempt
- GIVEN a user is in Activity 3
- WHEN the system displays a sextortion scenario
- THEN the user can identify the threat and manipulation tactics
- AND the user selects the correct response (don't pay, report, seek help)
- AND points are awarded based on accuracy

### Requirement: Psychological Manipulation Awareness
The system MUST present scenarios showing grooming and manipulation tactics.

#### Scenario: Grooming Pattern Recognition
- GIVEN a user is in Activity 4
- WHEN the system displays a manipulation scenario
- THEN the user can identify manipulation tactics (love bombing, isolation, secrecy demands)
- AND the user selects the correct defense
- AND points are awarded based on accuracy

### Requirement: Defense Scenario
The system MUST present a defense scenario integrating all themes.

#### Scenario: Integrated Defense
- GIVEN a user completes Activities 1-4
- WHEN the system displays the defense scenario
- THEN the user applies knowledge from all previous activities
- AND final score is calculated (200 points total, 70% threshold)
- AND results are displayed
